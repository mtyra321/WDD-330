• Error, Exceptions, and Warnings
  • system error: the system or other external device has a problem
  • programmer error: there is an error in the code
  • User error: the user inputted data or did something else wrong
  • Exceptions: an error that returns a value that can be used to deal with the error
  • stack trace: sequence of functions that lead to the point where the error occured
  • stack traces can help you find the source of the error
  • warnings: an error in the code that isn't enough to cause the program to crash
  • Browser Debugging
  • Alerts: shows a box, stops code from continuing
  • Alerts allow us to put breakpoints in the code
  • Console Debugging
  • Console.log() sends a message, doesn't stop the code
  • console.trace creates a stack trace within the console so you can see the path
  • Error Objects
  • Error objects: created by the host environment when an exception occurs
  • Error objects types: EvalError, RangeError, ReferenceError, SyntaxError, TypeError, URIError, and Internal Error
  • Error objects types: EvalError, RangeError, ReferenceError, SyntaxError, TypeError, URIError, and Internal Error
  • Use the throw statement to throw your own exception
  • Try catch:will try the code in the try part and catch and error in the catch part if there is an error
  • Finally block: added after catch, will do more code after the try and the catch
